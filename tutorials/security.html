Securing Your SSH Setup

Security Best Practices

  • Use Strong Passwords: Avoid default or weak passwords.
  • Change Default Port: Edit ~/.ssh/sshd_config and change: 
    Port 2222
  • Disable Password Login: Allow only key-based access for extra security.
  • Use SSH Keys:
    1. Generate key: ssh-keygen
    2. Copy key to server: ssh-copy-id user@host
  • Regularly Monitor Logs: Use logcat or check ~/.ssh for auth activity.
